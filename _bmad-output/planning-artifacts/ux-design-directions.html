IssueForge

Design Direction Showcase

Eight visual directions exploring different approaches to IssueForge's core clarification loop and issue review screens. Directions 1–6 show the clarification question step. Directions 7–8 show the post-generation review step. All are grounded in the established visual foundation: Forge Blue (#5B6CF0), Geist typeface, 4px base spacing, shadcn/ui components.

Evaluation criteria

Which layout matches how your users think? Which density feels right? Which persona (Sarah vs Marcus) does this serve best?

Light vs. Dark

Non-technical users default to light; developer users default to dark. Both modes are always available — these directions explore each.

What to look for

Visual weight of the question. Prominence of the sidebar. How the "why did you ask this?" disclosure is handled. Overall spaciousness.

Direction 1
Focused Chat — Light
Centred, immersive, no sidebar. The clarifying question is the only thing on screen. Maximum focus for non-technical users; minimal visual noise. Repo context is shown only in the header strip.

Direction 5
Zen — Light
Maximum whitespace. No sidebar, no stepper, just the question and an underline input. Progress is shown as three thin bars. Inspired by Typeform's focus-per-question approach. Feels calm and considered — could work well for non-technical users who find busy UIs anxiety-inducing.

Issue Review — Light

Fix: Subscription downgrade fails silently for enterprise accounts
## Description
When an enterprise account attempts to downgrade their subscription plan, the operation appears to succeed in the UI but the plan is not updated in the database. No error is shown to the user.
## Steps to Reproduce
1. Log in as an enterprise account owner
2. Navigate to Settings → Billing → Change Plan
3. Select a lower tier and confirm
4. Observe: success toast appears but plan remains unchanged
## Expected Behaviour
The subscription plan should be updated in the database and reflected in the UI within the same session.
